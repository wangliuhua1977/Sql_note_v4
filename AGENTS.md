# AGENTS.md

本文件约束整个仓库。

## 1. 项目目标

本仓库用于从 0 开始开发 **Sql_note_v4**：一个基于 **.NET 10 + WinUI 3** 的 Windows 桌面 SQL Notebook / PostgreSQL 客户端。

这是一次 **重写**，不是 Java 项目的逐文件迁移。允许重构内部实现，但必须保留核心产品能力，并重点增强：

1. 主窗体内多子窗口 SQL 编辑工作区
2. PostgreSQL-only（不再区分 Hive）
3. SQL 编辑器与联想引擎
4. 函数 / 存储过程工作台
5. 笔记管理中心
6. 独特笔记图标系统（稳定随机 + 可切换变体）
7. WinUI 3 主题管理与切换

## 2. 不可违反的硬约束

### 2.1 技术栈
- .NET 10
- WinUI 3
- MVVM
- Generic Host + Microsoft.Extensions.DependencyInjection
- SQLite（本地数据）
- WebView2（如编辑器宿主需要）
- Serilog（结构化日志）

### 2.2 产品约束
- 主交互必须是 **单主窗体 + 内部多文档子窗口**，不要改成以标签页为主的产品。
- 不再实现 Hive 类型分支，统一按 PostgreSQL 设计。
- 右侧任务/日志/Inspector 区域不做。
- 左侧保留对象浏览器；底部保留结果/消息/必要输出区。
- SQL 编辑器和联想引擎是第一优先级核心。
- 函数/存储过程工作台是第二优先级核心，必须支持历史版本、对比、发布、回滚、测试运行。
- 笔记管理必须是“管理中心”，不是简单 CRUD 列表。
- 笔记图标系统必须保留并增强，默认采用“稳定随机”。
- 必须支持 WinUI 3 主题管理与切换，至少包含浅色、深色、跟随系统三种正式主题；后续可扩展更多品牌主题。

### 2.3 工程约束
- 所有新代码必须先有清晰职责，再写实现。
- 禁止把大量逻辑重新塞进单个窗口类或 code-behind。
- View 只负责展示和绑定；业务流程必须进入 ViewModel / Service / Coordinator。
- 所有配置必须外置，禁止硬编码生产地址、token、AES 密钥、webhook、口令。
- SQLite 结构变更必须通过版本化 migration，不允许继续采用“启动时补洞”式演进。
- 涉及内容保存、版本保存、发布留痕的流程必须尽量单事务或一致性设计，避免“正文已写入但版本没记录”这类问题。

## 3. 目录与分层约束

建议解决方案至少拆为：
- src/SqlNote.App
- src/SqlNote.Protocol
- src/SqlNote.Data
- src/SqlNote.Metadata
- src/SqlNote.Editor
- src/SqlNote.Features
- tests/*

禁止把下列职责直接写入主窗体：
- 执行协议
- 元数据刷新
- 联想语义分析
- 结果集模型
- 笔记版本管理
- 例程发布/回滚
- 主题管理
- 图标生成

## 4. UI 与交互约束

### 4.1 主工作区
- 支持多个 SQL 子窗口并存。
- 子窗口支持：打开、激活、关闭、最大化、平铺、级联、恢复。
- 每个子窗口有自己的编辑器状态、执行状态、结果上下文。

### 4.2 SQL 编辑器
必须重点做好：
- 语法高亮
- PostgreSQL-only 语义联想
- alias / CTE / 子查询 / returning / group by / order by / join using 等上下文识别
- 查找替换
- 块执行
- 错误定位
- 格式化（可分阶段）
- 编辑器状态恢复

### 4.3 Routine Workbench
必须重点做好：
- 函数/过程打开
- 数据库源码拉取
- 本地草稿保存
- 历史版本保存
- 差异对比
- 发布
- 回滚
- 运行测试
- 错误回定位

### 4.4 笔记管理中心
必须支持：
- 搜索
- 过滤
- 列表 / 卡片视图
- 批量操作
- 收藏 / 置顶
- 本地版本入口
- 垃圾箱
- 当前会话
- 例程草稿统一管理
- 图标预览与切换

### 4.5 主题系统
必须支持：
- 跟随系统
- 明确浅色主题
- 明确深色主题
- 持久化用户主题偏好
- 主题切换后图标与关键 UI 组件视觉协调

## 5. 交付约束

每一轮任务都必须同时交付：
1. 修改文件清单
2. 设计说明
3. 编译验证结果
4. 关键交互验证结果
5. 未完成项与下一轮建议

## 6. 禁止事项

- 禁止一次性大补丁把整个项目“写完”。
- 禁止跳过编译验证就声称完成。
- 禁止为了省事把复杂 UI 逻辑塞进窗口代码后置解释。
- 禁止引入未经说明的重量级依赖。
- 禁止写死环境参数。
- 禁止擅自把多子窗口模式改成纯标签模式。

## 7. 开发节奏

必须按阶段推进：
- Phase 0: 仓库规约与骨架
- Phase 1: App 壳 + 分层骨架 + 主题系统 + 多子窗口工作区骨架
- Phase 2: SQL 编辑器 v1 + 结果区 v1
- Phase 3: 联想引擎 v1 + 元数据缓存 v1
- Phase 4: Routine Workbench v1
- Phase 5: 笔记管理中心 v1 + 图标系统 v1
- Phase 6: 增强与收口

除非文档明确允许，否则不得跨阶段提前大面积实现后续能力。
