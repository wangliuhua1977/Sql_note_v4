# 项目简报

## 项目名称
Sql_note_v4

## 产品定位
基于 .NET 10 + WinUI 3 的 Windows 桌面 SQL Notebook / PostgreSQL 客户端。

## 核心目标

### 核心 1：SQL 编辑器与联想引擎
这是第一优先级。

必须重点支持：
- PostgreSQL-only 语义联想
- 多子窗口编辑体验
- 错误定位
- 块执行
- 查找替换
- 后续支持格式化与模板

### 核心 2：Routine Workbench
函数 / 存储过程工作台必须做强。

必须支持：
- 源码拉取
- 草稿保存
- 历史版本
- 差异对比
- 发布
- 回滚
- 测试运行

### 核心 3：笔记管理中心
不是简单笔记列表，而是内容资产管理中心。

必须支持：
- 搜索
- 过滤
- 列表 / 卡片切换
- 批量整理
- 当前会话
- 例程草稿统一管理
- 垃圾箱
- 版本入口

### 核心 4：笔记图标系统
必须保留并增强。

要求：
- 默认稳定随机
- 同一笔记图标一致
- 可切换变体
- 美观、风格统一
- 与主题协调

### 核心 5：主题管理与切换
必须支持 WinUI 3 主题管理。

首期要求：
- Follow System
- Light
- Dark
- 持久化
- 主题切换生效于主工作区、子窗口、列表/卡片、图标系统

## 明确不做
- Hive 产品区分
- 右侧任务/日志/Inspector 区域
- 双工作模式并存（窗口模式与标签模式并存）

## 主界面结构
- 左侧：对象浏览器
- 中间：多子窗口工作区
- 底部：结果 / 消息 / 必要输出

## 第一阶段成功标准
- 工程骨架清晰
- 可编译
- 可启动
- 可切主题
- 可打开内部子窗口
- 代码层次没有塌到主窗体中
