# 分阶段验收标准

## 通用验收要求
每一轮都必须给出：
- 修改文件清单
- 设计说明
- 编译结果
- 关键交互验证
- 未完成项
- 下一轮建议

## Phase 0
目标：仓库规约层完成。

验收：
- `AGENTS.md` 存在
- `docs/` 中核心约束文档存在
- 阶段提示词存在
- 文档之间不冲突

## Phase 1
目标：App 壳 + 分层骨架 + 主题系统 + 多子窗口工作区骨架。

验收：
- 解决方案与项目结构建立完成
- WinUI 3 应用可编译、可启动
- 主窗体可打开
- 支持 Follow System / Light / Dark 三种主题切换
- 主题切换可持久化
- 可新建内部子窗口
- 子窗口支持激活、关闭、最大化
- 不存在把主业务塞进窗口 code-behind 的明显问题

## Phase 2
目标：SQL 编辑器 v1 + 结果区 v1。

验收：
- 子窗口中可用编辑器宿主可工作
- 支持文本输入、保存草稿、查找替换基础功能
- 支持块执行识别基础能力
- 结果区能渲染基础结果集
- 错误可展示

## Phase 3
目标：联想引擎 v1 + 元数据缓存 v1。

验收：
- 有 PostgreSQL-only 语境联想基础能力
- 支持表、列、函数、过程联想基础版本
- 元数据可缓存
- 元数据可刷新

## Phase 4
目标：Routine Workbench v1。

验收：
- 可打开 routine 源码
- 可保存 routine 草稿
- 可查看 routine 历史版本
- 可发布
- 可运行测试

## Phase 5
目标：笔记管理中心 v1 + 图标系统 v1。

验收：
- 笔记管理中心可搜索、过滤、列表/卡片切换
- 支持批量基础操作
- 图标系统生效且稳定随机
- 可切换图标变体
- 支持垃圾箱与版本入口

## Phase 6
目标：增强与收口。

验收：
- 例程差异对比/回滚完成
- 编辑器与联想增强完成主要项
- 工程结构稳定
- 关键流程可用
